# Repository Guidelines

## Project Structure & Module Organization

This is a Rust binary project for nannou creative-coding sketches.

- `Cargo.toml` defines the workspace members and shared profiles.
- `Cargo.lock` is committed for reproducible local runs.
- `rust/crates/main/main.rs` contains the thin nannou app runner, window setup, fullscreen shortcut, and persistence.
- `rust/crates/hot_reload/lib.rs` contains the hot-reloadable demo router, AppUI, DemoUI, FPS display, and demo list.
- `rust/crates/hot_reload/demo_01/` and `rust/crates/hot_reload/demo_02/` contain hot-reloadable sketch demos.
- `rust/crates/mcp_server/main.rs` contains a read-only MCP server built with the official Rust SDK.
- `scripts/main/` contains setup and primary development workflow scripts, including hot-reload runners.
- `scripts/other/` contains secondary utility scripts, including desktop, web, MCP, version, and export scripts.
- Do not add root-level PowerShell scripts under `scripts/`. Keep script implementations in exactly these two subfolders and update docs or callers to use the subfolder paths.
- `target/` is generated by Cargo and must remain untracked.

Place runner code under `rust/crates/main/` and hot-reloadable art code under snake_case demo folders in `rust/crates/hot_reload/`, such as `demo_03/`. Put images, fonts, shaders, or other runtime files under `assets/`.

## Build, Test, and Development Commands

- `cargo check` verifies compilation without producing a runnable binary.
- `cargo build` compiles the app.
- `cargo run -p nannou-creative-coding` launches the nannou window.
- `.\scripts\main\Install.ps1` installs or updates Rust through rustup.
- `.\scripts\other\RunDesktop.ps1` launches the app through the repository script.
- `.\scripts\main\RunDesktopWithHotReload.ps1` keeps the app running and hot reloads changes under `rust/crates/hot_reload/`.
- `.\scripts\other\RunMcpServer.ps1` runs the read-only MCP stdio server.
- `cargo fmt --check` verifies Rust formatting.
- `cargo fmt` applies standard Rust formatting.

Run commands from the repository root.

## Coding Style & Naming Conventions

Use standard Rust formatting via `rustfmt`. Prefer descriptive snake_case names for functions and variables, PascalCase for types, and SCREAMING_SNAKE_CASE for constants such as `WINDOW_WIDTH`.

Keep sketch constants near the top. Use conventional nannou lifecycle functions: `main` and window setup in `rust/crates/main/main.rs`, with hot-reloadable `update` and `view` functions in `rust/crates/hot_reload/lib.rs`.

## Testing Guidelines

There are no automated tests yet. For compile-level validation, run:

```powershell
cargo fmt --check
cargo check
```

If logic grows beyond simple drawing setup, add unit tests in the relevant Rust module or integration tests under `tests/`. Name tests after behavior, for example `centers_window_from_monitor_bounds`.

## Commit & Pull Request Guidelines

Use concise imperative commit messages, for example `Add centered nannou starter sketch`.

Pull requests should include a short change summary, verification run, and screenshots or visual notes for drawing/window changes.

## Agent-Specific Instructions

Keep changes scoped to this repository. Do not commit build output or local machine configuration. Honor the two-folder script layout: use `scripts/main/` for setup and primary development workflows, and use `scripts/other/` for secondary utility workflows. Do not recreate duplicate root-level script wrappers under `scripts/`; update README commands, docs, and callers to point at the canonical subfolder script instead. Prefer updating `scripts/other/RunDesktop.ps1` when the one-shot run workflow changes and `scripts/main/RunDesktopWithHotReload.ps1` when the hot-reload workflow changes.

Users may describe drawings and art in natural language. Translate requests into nannou sketches using the official guide at `https://guide.nannou.cc/`, especially drawing, color, geometry, animation, and app-structure patterns. Put sketch edits in the active snake_case demo folder under `rust/crates/hot_reload/` so `scripts/main/RunDesktopWithHotReload.ps1` can rebuild the hot library.

Use the internal names `AppUI` and `DemoUI` when discussing overlay changes with agents. Do not expose those labels in user-facing text. AppUI is the top shared overlay box and contains app-wide status such as FPS and `Keys: Left/Right`. DemoUI is the box below AppUI and contains the active demo name, `Memories: 000`, and demo-specific instructions such as mouse controls or `Input: None`. Left and Right arrow keys switch between compiled demos and recreate the selected demo state from scratch.

Window behavior belongs in `rust/crates/main/main.rs`. `F` toggles fullscreen and the runner persists fullscreen state, monitor, position, and size to `target/window-state.json`.

MCP helper behavior belongs in `rust/crates/mcp_server/main.rs`. Keep it read-only unless the user explicitly asks for write-capable MCP tools.

## Git Safety

Non-destructive Git operations are allowed for normal repository work. This includes `git init`, creating a new remote repository when explicitly requested, adding remotes, creating branches, staging, committing, fetching, and normal non-force pushes.

Destructive Git remains prohibited. Do not rewrite history, discard local changes, delete branches or tags, force-push, run `git reset --hard`, run `git clean`, rebase, amend, squash, or delete repositories.

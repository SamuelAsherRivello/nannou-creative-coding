# Repository Guidelines

## Project Structure & Module Organization

This is a Rust binary project for nannou creative-coding sketches.

- `Cargo.toml` defines the package and dependencies.
- `Cargo.lock` is committed for reproducible local runs.
- `src/main.rs` contains the current nannou app entrypoint, window setup, and drawing code.
- `scripts/Run.ps1` runs the app from the repository root.
- `target/` is generated by Cargo and must remain untracked.

Place new Rust modules under `src/`. Put images, fonts, shaders, or other runtime files under `assets/` and load them relative to the project root.

## Build, Test, and Development Commands

- `cargo check` verifies compilation without producing a runnable binary.
- `cargo build` compiles the app.
- `cargo run` launches the nannou window.
- `.\scripts\Run.ps1` launches the app through the repository script.
- `cargo fmt --check` verifies Rust formatting.
- `cargo fmt` applies standard Rust formatting.

Run commands from the repository root.

## Coding Style & Naming Conventions

Use standard Rust formatting via `rustfmt`. Prefer descriptive snake_case names for functions and variables, PascalCase for types, and SCREAMING_SNAKE_CASE for constants such as `WINDOW_WIDTH`.

Keep sketch constants near the top of the file. Use conventional nannou lifecycle functions: `main`, `model`, `view`, and focused helpers such as `center_window`.

## Testing Guidelines

There are no automated tests yet. For compile-level validation, run:

```powershell
cargo fmt --check
cargo check
```

If logic grows beyond simple drawing setup, add unit tests in the relevant Rust module or integration tests under `tests/`. Name tests after behavior, for example `centers_window_from_monitor_bounds`.

## Commit & Pull Request Guidelines

This repository has no commit history yet, so no existing convention is established. Use concise imperative commit messages, for example `Add centered nannou starter sketch`.

Pull requests should include:

- A short summary of the change.
- Verification run, especially `cargo fmt --check` and `cargo check`.
- Screenshots or a brief visual note for drawing/window changes.

## Agent-Specific Instructions

Keep changes scoped to this repository. Do not commit generated build output or local machine configuration. Prefer updating `scripts/Run.ps1` when the normal run workflow changes.

Users may describe desired drawings and art results in natural human language. Translate those requests into nannou sketches by leaning on the concepts and examples in the official guide at `https://guide.nannou.cc/`, especially its drawing, color, geometry, animation, and app-structure patterns. When a request is visually ambiguous, choose a simple, idiomatic nannou implementation and keep the result easy to iterate.
